# 安全策略

感谢你帮助让 ShareGPT 更安全。本项目内嵌第三方 AI 网页、在本机开启代理端口、并由服务端下发网络配置，属于**安全敏感项目**，我们非常重视任何与安全相关的报告。

## 请勿公开披露

**不要在公开 issue、PR、讨论区或社交媒体里报告安全漏洞。** 公开披露会让尚未修复的问题被滥用，危及所有使用者。

## 如何上报

请通过以下私密渠道之一报告：

- **GitHub Security Advisories（推荐）**：在仓库 `Security` → `Report a vulnerability` 提交私密报告。
- **邮件**：`security@example.com`
  > 维护者请替换为真实邮箱；或直接启用 GitHub Security Advisories 作为唯一私密渠道。

报告时请尽量包含：

- 受影响的组件（客户端 / `collab_server2/` 协作服务端 / `admin_console/` 管理控制台）与版本号。
- 漏洞类型与影响（能做什么、危害范围）。
- 详细复现步骤、最小化的复现环境，必要时附 PoC。
- 你认为可能的修复方向（可选）。

## 属于安全报告范围的内容

除常规漏洞外，以下**代理与配置链路相关**的问题请务必上报：

- **内嵌第三方网页的权限**：内嵌 ChatGPT / Claude / Gemini 页面可越权访问本机资源、IPC 通道、`preload` 暴露的 API、读取其他标签会话或登录态等。
- **本地 SOCKS / 代理端口**：sing-box 等本机监听端口被非预期进程访问、未做本地绑定、可被远程连接、可被用作开放代理或流量中转。
- **配置下发链路**：协作服务端向客户端下发 Sender 默认配置 / 机场节点 / 更新清单的过程中，存在中间人篡改、未校验来源、配置注入、把恶意节点或恶意更新源下发给客户端等风险。
- **更新机制**：基于 GitHub Releases 的自动更新（`latest.yml`、安装包）被替换、降级或绕过校验。
- **密钥与凭据**：`DEV_TOKEN`、用户凭据、聊天数据、登录态在传输或存储中的泄露。
- 认证绕过、越权访问其他用户的聊天 / 文件 / 统计数据，以及服务端的注入 / 路径穿越等问题。

## 支持的版本

| 版本                   | 是否提供安全修复  |
| ---------------------- | ----------------- |
| 1.0.x（当前正式版）    | ✅                |
| 5.x 及更早（测试版本） | ❌ 请升级到正式版 |

## 预期响应时间

- **3 个工作日内**确认收到报告。
- **7 个工作日内**给出初步评估与处理计划。
- 修复时间视严重程度而定；高危问题会优先处理，并在修复发布后与你协调披露时间。

我们欢迎并感谢负责任的披露。在你同意的前提下，我们会在修复说明中致谢。
