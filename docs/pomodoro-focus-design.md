# 番茄钟 / 专注 + 团队排名 设计（调研 + 方案，待确认）

> 你的需求：加一个番茄钟组件；能在团队里对相关数据排名；先调研设计。2026-06-23。

## 1. 调研结论（best-in-class 番茄钟）
参考 Forest / Focus To-Do / Pomofocus / TickTick：
- **计时核心**：专注(默认25m) / 短休(5m) / 长休(15m，每 4 个番茄后)；三者时长**各自可调**；自动开始下一段(可关)。
- **大号圆环倒计时** + 开始/暂停/重置/跳过；阶段切换桌面通知(本应用已有 `showSystemNotification`)。
- **任务绑定**(TickTick/Focus To-Do 式)：可选绑定一个待办任务，统计「每个任务花了几个番茄」——正好接本应用现有 **待办模块**。
- **统计反馈闭环**：今日专注分钟数 / 番茄数 / 连续天数(streak) / 周视图。
- **可选氛围音**(雨声/白噪音)——锦上添花，体积权衡后再定。
- **游戏化/社交**(Forest 一起种树、Focusmate)：对应你要的「团队排名」。

来源：[Reclaim 11 款番茄钟](https://reclaim.ai/blog/best-pomodoro-timer-apps)、[Zapier best pomodoro](https://zapier.com/blog/best-pomodoro-apps/)。

## 2. 团队排名（复用本应用现成模式）
本应用已有 **「使用统计」面板**：按群把每个用户的 GPT 用量做排行(服务端聚合 `buildServiceUsageStats` + stats 接口)。**专注排名照搬这套**：
- 客户端每完成一个专注段 → 上报 `{username, minutes, count, ts}` 到协作服务器(按群)。
- 服务端按区间(今日/本周)聚合每人专注分钟/番茄数 → 返回**群内排行榜**。
- 面板展示：头像 + 昵称 + 专注时长/番茄数 + 名次(像现有用量排行)。

## 3. 方案

### 放在哪（待你定）
- **A. 新建左侧导航「专注」**：计时器 + 个人统计 + 团队排行，独立清爽。
- **B. 并入现有「待办/备忘」模块**：加一个「专注」分页(TickTick 式，和任务绑定最顺)。

### 数据 / 同步
- 本地：`focus.json`(或并入 tasks.json) 存专注会话 `{id,start,minutes,taskId?,tag?}`；个人统计本地算。
- 团队排名：**需要服务端聚合**(因为要跨用户)。两条路：
  - 复用 user-store 不行(那是按用户隔离，看不到别人)。
  - **需在协作服务器加一对端点**：`POST /api/focus/report`(上报会话) + `GET /api/focus/leaderboard?range=today|week`(群内排行)。这是一次**服务端 graft + 部署到 3 群**(和之前 user-store/team-calendar 同套路)。
  - ⚠️ 这会**触碰生产服务器**——按你之前「先本地测」的习惯，我可以先在本地服务器验证，确认后再部署到群。

### 计时器实现要点
- 计时用 `performance.now()`/绝对时间戳算，避免后台 throttle 不准；切到后台仍准确。
- 单实例：全局只一个计时器(放 zustand store，跨面板可见，标题栏可显剩余时间)。
- 阶段结束：本应用 `showSystemNotification` + 提示音。
- 完成一个专注段才计入统计(中途放弃不计)。
