# ShareGPT 前端重构 · 目标文档（待你审阅/补充）

> 分支 `feature/ui-rebuild`（基于 4.2.0 `main` d263e54，**main 钉死不动**）。

## ✅ 已确认决定（2026-06-12）
- **组件栈**：shadcn/ui（拷贝式、完全拥有、避免框架通用脸）
- **语言**：TypeScript
- **范围**：**先做核心面板打样** = 外壳(顶栏+侧导航+主题) + 登录 + 发送/接收服务 + 聊天；GPT/Gemini/统计/日志/设置 后续迭代
- **设计参考**：**Telegram 风格**——参考其布局与交互（左侧会话/导航栏 + 主内容区、圆角气泡、清爽留白、单强调色），**但不照搬、不做四不像**，保持本产品自身识别（琥珀强调色）
- **profile**：保留独立窗口
- **主栈**：electron-vite/Vite + React 19 + TS + Tailwind v4 + shadcn/ui + Zustand + lucide-react

---

## 0. 为什么要做（问题陈述）

上一轮「渐进精修」本质只是**统一了配色 + 理了 token**，DOM 结构、布局、组件体系原封不动 → 视觉「四不像」。
本次目标是**真正重构前端**：引入现代 UI 框架与组件体系，重写界面结构与交互，而非再调色。

---

## 1. 技术栈选型（我的推荐 + 备选，待你拍板）

**推荐主栈：`electron-vite` + React 19 + TypeScript + Tailwind CSS v4 + shadcn/ui**

| 层 | 选型 | 理由 |
|---|---|---|
| 构建 | **electron-vite** | 2026 Electron 事实标准, 极快 HMR, 主/预加载/渲染三段式构建 |
| 框架 | **React 19 + TS** | 市占 ~68%, 生态最大, 类型安全, 长期可维护 |
| 样式 | **Tailwind CSS v4** | 免配置(Vite 插件), 设计 token 一致, 工具类高效 |
| 组件 | **shadcn/ui** | 代码拷进项目你**完全拥有**(无版本锁/无"框架脸"), 现代且可深度定制——最能避免"四不像" |
| 状态 | Zustand（轻量）/ TanStack Query（异步） | 替代当前散落的全局变量 |
| 图标 | lucide-react | 与 shadcn 配套 |

**备选（如果你更倾向）：**
- **Mantine**：开箱组件最多（表单/通知/弹窗/日期/富文本/图表都自带），上手最快，适合"快速堆出完整功能"。
- **Ant Design**：数据密集型企业风、表格/表单最强、中文生态好；但偏"通用后台脸"，定制成本高。

> 这三套的视觉/工作量差异很大，是**本次最关键的一个决定**，见文末「待拍板」。

---

## 2. 范围（改什么 / 不改什么）

**重写（渲染层 `src/renderer`）：**
- 全部 UI 结构（`index.html` 单文件 → React 组件树）
- 全部界面交互逻辑（`renderer.js` ~6200 行 → 拆成组件 + hooks + store）
- 样式体系（`styles.css` ~4500 行 → Tailwind + shadcn 设计 token）
- 独立 profile 窗口（`profile.html`/`profile.js`）一并重写

**基本保留（主进程 / 契约）：**
- `src/main/backend.js`、`appFactory.js` 的业务逻辑（sing-box/frpc 监管、IPC 处理、supervisor 自愈等）
- **`preload.js` 的 IPC 契约**（`window.electronAPI.*`）——这是新渲染层与主进程的边界，**逐一保留/对齐**，新 UI 只换"壳"不换"管道"
- 数据文件格式（`settings.json` / `chat_history.json`）——**100% 兼容**，不丢历史数据

**需配套小改主进程：**
- 渲染层加载方式：dev 走 Vite dev server URL，prod 走构建产物（改 `appFactory.js` 的 loadURL/loadFile + 打包脚本）
- 保留 dev 数据隔离开关 `SHAREGPT_USER_DATA`（已在另一分支验证，移植过来）

## 3. 必须 1:1 保留的功能清单（功能对等基线）

> 重构后**功能不能少一个**。这是验收的硬底线。

**窗口/外壳**
- [ ] 三模式 `--mode=sender|receiver|all`
- [ ] 自定义顶栏（品牌、连接状态、**日/夜主题切换**、窗口控制键、可拖拽区）
- [ ] 侧边导航（多面板切换 + 三态高亮）
- [ ] 首次使用引导卡（setup-guide，可关闭、幂等）
- [ ] 服务异常/自愈横幅（supervisor banner）

**面板**
- [ ] 发送服务（节点/本地端口/回退方式/目标域名 表单 + 启动/停止）
- [ ] 接收服务（frps/vmess/转发 表单 + 启停）
- [ ] 日志（操作日志 + 接收端日志，等宽字体可读）
- [ ] 账户/登录（服务地址/账号/密码/记住密码 + 登录态）
- [ ] 协作聊天（联系人/最近会话侧栏、消息气泡、附件**剪贴板粘贴+文件**、在线用户、未读/通知、置顶）
- [ ] 消息设置
- [ ] 统计（查询数饼图、用户排行、时间区间预设）

**AI 网页（关键架构约束）**
- [ ] ChatGPT / Gemini 用**原生 `WebContentsView`**（主进程管理），渲染层提供**宿主容器 div**并把其 bounds 同步给主进程定位；多标签会话、新建标签、host 白名单、查询计数统计——**全部保留这套机制，不可改成 iframe**

**数据/实时**
- [ ] 设置持久化、聊天历史持久化（格式兼容）
- [ ] 代理热更新（WS `proxy_update`）、协作消息实时推送

## 4. 设计方向（视觉目标）

- **现代桌面应用质感**：清晰层级、克制留白、一致圆角/间距刻度、流畅微交互。
- **品牌色**：沿用琥珀 `#f59e0b` 作单一强调色（上一轮已确立），深/浅双主题。
- **参考标杆**：Linear（信息密度+克制）、Vercel/shadcn（中性高对比+单强调）、Mihomo Party / Clash Verge（代理工具同类最佳实践）。
- **目标**：不是换皮，而是**重排信息架构与组件**——表单、状态、导航、聊天、统计都用框架组件库重建，达到"一眼现代"。

---

## 5. 硬约束（不可违反）

1. **绝不影响正在运行的生产客户端**（`sharegpt-sender-4.2.0`）：dev 用隔离数据目录，不杀其进程、不抢端口。
2. **git 可回退**：`main` 钉死；重构在 `feature/ui-rebuild`，分阶段 commit。
3. **数据兼容**：升级后老的 settings/聊天记录**不丢、可读**。
4. **功能对等**：第 3 节清单逐项不可缺失。
5. **IPC 契约保留**：`preload` API 不破坏（主进程零改业务逻辑）。

---

## 6. 迁移策略（安全并行）

- **新旧并存**：新 React 渲染层独立构建到 `src/renderer-next`（或新目录），**不删旧 `src/renderer`**；用环境变量/flag 切换主进程加载新旧渲染层，便于随时对比/回退。
- **分阶段验证**：每个面板迁完即对照旧版走查功能，确认对等再进下一个。
- **最后切换**：全部对等 + 打包验证通过后，才把默认入口切到新渲染层；旧版保留一个版本周期。

## 9. 风险

- **WebContentsView 宿主定位**：React 重排后宿主 div 的 bounds 同步需精确（resize/滚动/标签切换时机），是最易出问题处 → 优先打通 M4 的最小验证。
- **6200 行逻辑迁移**：体量大，按面板切片、保留 IPC 契约可控；并行开发需文件隔离防冲突。
- **打包链路**：electron-vite 产物路径与现有 electron-builder 配置对接需调试。
